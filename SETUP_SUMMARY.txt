If needed, you can customize in `.env`:

```ini
# PDF Configuration
FILE_PATH=Egyptian_ERP_System_Comprehensive_Manual.pdf

# LM Studio Settings
LM_STUDIO_BASE_URL=http://localhost:1234/v1
LM_STUDIO_MODEL=llama-3.2-3b-instruct
TEMPERATURE=0.3

# Performance Tuning
CHUNK_SIZE=600        # Document chunk size
SIMILARITY_K=10       # Number of docs to retrieve

# Server Settings
FLASK_PORT=5000
FLASK_HOST=0.0.0.0
```

## 🆘 Troubleshooting

### "PDF not found"
- ✅ Copy `Egyptian_ERP_System_Comprehensive_Manual.pdf` to the folder
- ✅ Ensure filename matches exactly

### "LM Studio not responding"
- ✅ Make sure LM Studio is running
- ✅ Check: http://localhost:1234
- ✅ Verify a model is loaded

### "Port 5000 already in use"
- ✅ Edit `.env`: Change `FLASK_PORT=8000`

### "Module not found"
- ✅ Activate virtual environment
- ✅ Run: `pip install -r requirements.txt`

## 🎓 Understanding the Flow

1. **You run** `start.bat` or `start.sh`
2. **Script checks** Python, PDF, creates venv
3. **Script installs** dependencies (from requirements.txt)
4. **Server starts** and loads the PDF
5. **Creates vector index** for fast search
6. **API is ready** at http://localhost:5000
7. **You can test** with `test_api.py`
8. **Users query** via `/ask` endpoint

---

## 🔄 What Happens When You Ask a Question

1. User POSTs: `{"question": "What is sick leave?"}`
2. Server retrieves similar chunks from PDF
3. Server sends chunks + question to LM Studio
4. LM Studio generates answer
5. Server returns: `{"answer": "...", "sources": [5, 8, 10]}`
